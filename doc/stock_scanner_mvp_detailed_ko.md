# 주식 스윙 스캐너 웹앱 MVP 상세 스펙 문서

## 1. 문서 목적

이 문서는 **코스피 스윙 매매용 자동 종목 발굴 웹앱**의 MVP 범위를 정의한다.  
목표는 사용자가 매일 차트를 수십 개씩 수동으로 보지 않아도, 본인이 원하는 조건에 맞는 종목을 자동으로 찾아보고 저장하고 복기할 수 있게 만드는 것이다.

이 문서는 아래를 포함한다.

- 제품 목적
- 사용자 시나리오
- 매매 조건 정의
- 기능 요구사항
- 비기능 요구사항
- 시스템 구조
- 데이터 모델
- API 설계 초안
- 스캔 로직
- 배포 및 운영 방식
- MVP 이후 확장 방향

---

## 2. 제품 한 줄 정의

**상승 추세 안에서 눌렸다가 다시 반등하려는 코스피 종목을, 사용자가 설정한 조건으로 자동 탐색하고 점수화해서 보여주는 개인용 웹앱**

---

## 3. MVP 목표

### 3.1 핵심 목표
사용자가 아래 과정을 웹에서 혼자 할 수 있어야 한다.

1. 회원가입 / 로그인
2. 나만의 전략 생성
3. 전략 조건 수정
4. 스캔 실행 또는 자동 스캔 결과 확인
5. 조건에 맞는 종목 리스트 확인
6. 종목별 세부 이유 확인
7. 관심종목 저장
8. 간단한 매매일지 기록

### 3.2 MVP에서 반드시 해결해야 하는 문제
현재 사용자의 문제는 다음과 같다.

- 종목을 어떻게 발굴해야 할지 막막함
- 차트를 하나씩 다 보기 귀찮음
- 눌림목 기준을 기계적으로 적용하고 싶음
- HTS 없이 웹 기반으로 쓰고 싶음
- 조건을 나중에 자유롭게 바꾸고 싶음
- 매매일지와 연결해 내 전략이 맞는지 보고 싶음

### 3.3 MVP 성공 기준
아래가 가능하면 MVP 성공으로 본다.

- 사용자가 웹에서 전략을 저장할 수 있다.
- KOSPI 종목 전체를 기준으로 자동 스캔할 수 있다.
- 결과 종목이 조건 통과 사유와 함께 보인다.
- 결과를 점수순으로 정렬할 수 있다.
- 결과 종목을 관심종목으로 저장할 수 있다.
- 간단한 매매일지를 쓸 수 있다.

## 6. 매매 전략 정의

## 6.1 전략의 본질
이 전략은 **완전한 바닥잡기**가 아니다.  
이미 어느 정도 살아 있는 흐름 안에서, **잠깐 눌렸다가 다시 튀는 자리**를 찾는 전략이다.

쉽게 말하면 아래와 같다.

1. 원래 어느 정도 괜찮던 종목이
2. 단기적으로 눌렸고
3. 너무 과하게 밀린 신호가 나오며
4. 다시 반등 시작 신호가 뜨는 시점을 찾는다.

## 6.2 현재 기준 전략 조건
아래는 현재 사용자가 원하는 기본 전략 조건이다.

### 필수 조건
1. 시장은 **코스피만**
2. 일정 시가총액 이상 종목만 대상
3. 거래대금이 충분한 종목만 대상
4. 볼린저밴드 기준, 가격이 하단 부근에 있어야 함
5. RSI는 약한 구간에 있어야 함
6. **RSI(14)가 RSI 시그널(9)을 아래에서 위로 뚫는 순간** 또는 그 직후여야 함
7. 가격은 **20일선 위**이거나 최소한 20일선 근처에서 버티는 모습이어야 함
8. 최근 2~3일 외국인 순매수가 들어오는 종목 선호
9. 보유 기간은 수일 ~ 최대 2주
10. 손절 기준은 대략 -3%

### 이동평균선 해석
- **20일선:** 가장 중요한 기준선. 이 선 아래로 너무 무너진 종목은 제외하는 방향
- **5일선:** 단기 흐름 확인용. 5일선이 20일선 위거나 가까우면 더 좋음
- **60일선:** 참고용. 장기 추세 확인용이지만 MVP에서는 필수 조건 아님

### RSI 해석
- 단순히 RSI가 낮다고 매수하지 않음
- 핵심은 **RSI(14)가 RSI 시그널(9)을 상향 돌파하는 타이밍**
- 즉, '많이 빠진 상태'보다 '다시 고개 들기 시작하는 순간'이 중요함

### 볼린저밴드 해석
- 볼린저밴드 설정: **20일, 표준편차 2**
- 가격이 하단에 닿았거나 하단 근처에서 버티는 종목 선호
- 하단을 살짝 이탈했다가 회복하는 패턴도 감시 가능

## 7. 제품 차별화 포인트

이 제품은 단순 조건검색기가 아니라, 아래 기능으로 HTS와 차별화한다.

### 7.1 전략 템플릿
사용자가 복잡한 수식을 직접 만들지 않고,
- 눌림목 반등형
- 하단 밴드 반등형
- 외인 순매수 결합형
처럼 전략 단위로 시작 가능

### 7.2 점수화
조건 충족 여부만 보여주는 게 아니라,
**왜 이 종목이 더 좋은 후보인지 점수와 근거로 보여줌**

### 7.3 이유 표시
각 종목마다 아래 같은 이유를 보여줌

- RSI 상향 돌파
- 볼밴 하단 근접
- 20일선 위
- 외인 3일 순매수
- 거래대금 통과

### 7.4 매매일지 연결
스캐너 결과와 실제 매매 기록을 연결해
어떤 조건이 실제 성과가 좋은지 나중에 볼 수 있게 함

### 7.5 웹 기반 단순 UX
HTS 없이 웹에서 설정 / 저장 / 결과 확인 / 복기가 가능해야 함

## 9.3 종목 스캔 기능
### 대상
- 코스피 종목 전체
- 사용자 설정에 따라 시총, 거래대금 필터 적용

### 스캔 방식
- 일봉 기준 EOD 스캔이 기본
- 선택적으로 장중 스캔 가능
- 장중 스캔은 MVP에서는 5분 단위 또는 10분 단위 선택 가능하게 단순화

### 계산 항목
각 종목에 대해 아래를 계산한다.
- 최근 가격
- 5일선
- 20일선
- 60일선
- 볼린저밴드 상단/중앙/하단
- RSI(14)
- RSI 시그널(9)
- 외국인 최근 N일 순매수 합계
- 거래대금
- 조건 충족 여부
- 최종 점수

## 9.6 점수화 시스템
MVP에서는 직관적인 가중치 기반 점수를 사용한다.

### 예시 기본 점수
- RSI 상향 돌파: +30
- RSI 값이 목표 구간(예: 30~40): +15
- 볼밴 하단 근접: +20
- 가격이 20일선 위: +15
- 5일선이 20일선 위: +10
- 외인 N일 순매수: +20
- 거래대금 조건 충족: +10

### 점수 결과
- A등급: 80점 이상
- B등급: 60~79점
- C등급: 40~59점
- 제외: 39점 이하 또는 필수조건 불충족

### 필수조건 예시
아래는 점수 이전에 선 필터링 가능
- 코스피
- 시총 조건
- 거래대금 조건
- RSI 교차 여부
- 20일선 근처 여부

## 9.8 알림 기능
MVP에서는 너무 복잡하지 않게 간다.

### 필수

- 새 조건 충족 종목 알림

### 채널
- 웹 내 알림
- 이메일 또는 텔레그램 중 1개 이상 선택 구현

### 알림 내용 예시

---

## 9.9 매매일지 기능
매매일지는 최소형으로 설계한다.

### 입력 항목
- 날짜
- 종목
- 매수 이유
- 수량
- 진입가
- 매도가
- 수익률
- 수익
- 비고

### 자동 계산
- 수익 = (매도가 - 진입가) × 수량
- 수익률 = (매도가 - 진입가) / 진입가

### 연결 포인트
- 스캐너 결과에서 바로 일지 작성 가능
- 나중에 어떤 전략 결과로 진입했는지 연결 가능

## 11.1 전체 구조
- 프론트엔드: Next.js
- 백엔드: FastAPI
- DB: PostgreSQL
- 스케줄러: cron 또는 background worker
- 데이터 소스: 한국투자증권 Open API
- 서버: Mac mini self-hosted
- 리버스 프록시: Nginx 또는 Caddy
- 컨테이너: Docker

## 11.2 구조 흐름
1. 스케줄러가 정해진 시간에 실행
2. 백엔드가 종목 리스트 조회
3. 종목별 시세 / 수급 데이터 수집
4. 지표 계산
5. 전략별 조건 적용
6. 결과 저장
7. 사용자 웹에서 결과 조회
8. 알림 조건 맞으면 알림 발송

## 23. 요약

이 MVP는 단순 조건검색기를 만드는 것이 아니다.

이 제품의 핵심은 아래 5가지다.

1. **내 전략을 웹에서 저장하고 바꿀 수 있어야 한다**
2. **코스피 종목을 자동으로 스캔해야 한다**
3. **왜 이 종목이 후보인지 이유와 점수로 보여줘야 한다**
4. **관심종목과 매매일지로 실제 투자 흐름과 연결되어야 한다**
5. **HTS 없이 웹에서 직관적으로 쓸 수 있어야 한다**
